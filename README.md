# AI Food Ordering Chatbot

A fully functional AI-powered food ordering chatbot built with **Streamlit**, **LangChain**, **Groq LLM**, **ChromaDB (RAG)**, and **SQLite**.

## Quick Start

### 1. Create Virtual environment and Install Dependencies

```bash
cd food_bot
python -m venv venv
.\venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Set Your Groq API Key

```bash
cp .env.example .env
# Edit .env and replace the placeholder with your real key
# Get a free key at: https://console.groq.com
```

### 3. Generate the Menu PDF if menu already not in the data folder

```bash
python generate_menu_pdf.py
# Creates data/menu.pdf — the source document for RAG
```

### 4. Launch the App

```bash
streamlit run app.py
```

The app will automatically:
- Create the SQLite database (`data/orders.db`)
- Build the Chroma vectorstore from `data/menu.pdf`
- Start the LangChain agent

## Architecture & Key Components

### RAG (Retrieval-Augmented Generation)

| Component | Detail |
|-----------|--------|
| **Source** | `data/menu.pdf` (generated by `generate_menu_pdf.py`) |
| **Embeddings** | `all-MiniLM-L6-v2` via HuggingFace (free, local, no API key) |
| **Vectorstore** | Chroma, persisted to `vectorstore/chroma_db/` |
| **Retriever** | Top-8 semantic matches per query |
| **Purpose** | Provides accurate menu info, prices, and descriptions |

The `menu_search` LangChain Tool wraps the retriever, allowing the agent to query the menu in natural language before answering any food-related question.

### LangChain Agent

The agent uses **tool-calling** with:

| Tool | Purpose |
|------|---------|
| `menu_search` | RAG retrieval over the menu PDF |
| `place_order` | Writes a confirmed order to SQLite |
| `get_order` | Looks up an existing order by ID |

Built with `create_tool_calling_agent` + `AgentExecutor`. Chat history is passed as `MessagesPlaceholder` for multi-turn awareness.

### Custom Database Tools (`tools/order_tool.py`)

Two **custom `BaseTool` subclasses** handle all database interaction:

- **`PlaceOrderTool`** — accepts structured items, resolves prices from the built-in price map, calls `utils/database.py::create_order()`, returns a markdown order confirmation.
- **`GetOrderTool`** — fetches an order by ID and returns a formatted summary.

## Example Interactions

| User says | What happens |
|-----------|-------------|
| *"What's on the menu?"* | Agent calls `menu_search` → returns RAG chunks from menu PDF |
| *"Tell me about the Cheese burst pizza"* | Agent calls `menu_search("Cheese burst pizza")` → returns description & price |
| *"Order 2 Margherita pizzas and a Milkshake"* | Agent calls `place_order` → saves to SQLite → confirms with order ID |
| *"Get order #5"* | Agent calls `get_order(5)` → returns order details |
